10 главных выводов

1

2 из 5 Tier 1 поглощены в 2025: Paradox → Workday ($1B), Moveworks → ServiceNow ($2.85B). Консолидация платформ — основной путь выхода для AI HR point solutions.

2

Заявления AUTOMATE систематически завышены: только Paradox (scheduling/screening) и Leena AI (тикеты) обеспечивают измеримую end-to-end автоматизацию; Eightfold и Beamery — AUGMENT.

3

Per-employee ценообразование побеждает в enterprise: Leena AI, Eightfold, Moveworks берут оплату за сотрудника, выравнивая доход вендора с headcount клиента.

4

Ни один стартап не предлагает on-prem: все 10 — SaaS-only. Критический GAP для российского enterprise и регулируемых индустрий.

5

Rippling — PLG-аномалия: единственный PLG-first, $570M+ ARR, 10 255 отзывов на G2, оценка $16.8B — принципиально другой GTM.

6

Beamery подаёт сигналы бедствия: два раунда сокращений в 2023 (12% + 25%), $223M привлечено при ACV $220K–$580K в сжимающемся рынке.

7

Сертификаты ≠ безопасность: Paradox имел SOC 2 Type II + ISO 27001, когда тривиальная уязвимость (пароль «123456») раскрыла 64M записей кандидатов.

8

Eightfold — лидер по compliance: FedRAMP Moderate + DISA IL4 + ISO 42001 (AI management), но G2 рейтинг (8.3) самый низкий среди Tier 1.

9

Multi-LLM — новый стандарт: Leena AI открыто использует WorkLM + Gemini + Claude + OpenAI динамически; большинство скрывают LLM-стек.

10

Самый конкурентный ICP: enterprise 1K–10K сотрудников, покупатель CHRO — 4+ стартапа борются за одного клиента.

Стратегия: On-Prem HR Copilot для RU рынка

Детальная проработка модулей для заимствования. Что взять у каждого лидера, как адаптировать для российского enterprise.

Модуль 1 — Multi-LLM Agentic архитектура (по модели Leena AI)

🧠

Центральный оркестратор + доменные агенты

Источник: Leena AI Agentic Architecture

Leena AI использует центральный оркестратор, который маршрутизирует запросы к специализированным агентам по доменам (HR, IT, Finance). Каждый агент имеет свой контекст, знания и набор инструментов.

  • Что копировать: Паттерн «оркестратор + специализированные агенты» — каждый агент отвечает за свой домен (кадровый учёт, подбор, обучение, расчёт зарплат)
  • Адаптация для RU: Заменить WorkLM на fine-tuned Mistral/Llama на российском HR-корпусе (ТК РФ, кадровые документы, ЛНА)
  • Агенты для MVP: (1) HR Service Desk агент — ответы на вопросы по политикам, (2) Кадровый агент — действия в 1С:ЗУП, (3) Рекрутинг агент — скрининг и scheduling
Сложность: высокая Влияние: критичное Срок: 4-6 мес. MVP
🔀

Динамический роутинг между LLM

Источник: Leena AI WorkLM + Gemini + Claude + OpenAI

Leena AI динамически выбирает LLM в зависимости от задачи: WorkLM для типовых HR-запросов, Claude для сложного reasoning, Gemini для мультимодальных задач, OpenAI для low-latency.

  • Что копировать: Роутинг по критериям: сложность, чувствительность данных, latency requirements, стоимость
  • Стек для RU: YandexGPT/GigaChat (русский язык, закрытые API) + Mistral/Llama (open-source, on-prem) + fine-tuned модель на HR-домене
  • Ключевое отличие: Все модели должны работать on-prem или в private cloud — без внешних API для чувствительных данных (персданные сотрудников)
  • Fallback chain: Локальная модель → Private cloud модель → Эскалация на человека
Сложность: высокая Влияние: критичное Срок: 3-4 мес.

Модуль 2 — Conversational Hiring Automation (по модели Paradox)

💬

AI-рекрутер для массового найма

Источник: Paradox Olivia — McDonald's McHire (масштаб: 60+ стран)

Paradox доказала на масштабе McDonald's: conversational AI сокращает time-to-hire с 3 дней до 3 минут для массовых позиций. Ключевое — автоматизация scheduling, screening, FAQ в одном чат-интерфейсе.

  • Что копировать: Полный цикл «отклик → скрининг-вопросы → назначение интервью → напоминание → onboarding документы» через мессенджер
  • Каналы для RU: Telegram Bot API (основной), WhatsApp Business, VK, корпоративные мессенджеры (Яндекс.Мессенджер, VK Teams, eXpress)
  • Интеграции: hh.ru API для получения откликов, 1С:ЗУП для оформления, Google/Яндекс Calendar для scheduling
  • Чего избегать: Не обещать «90% автоматизации» — честно позиционировать как автоматизацию рутины, а не замену рекрутера
Сложность: средняя Влияние: высокое Срок: 2-3 мес. MVP
📋

Conversational Pre-Screening + Assessments

Источник: Paradox Traitify + Olivia screening flow

Paradox приобрела Traitify для визуальных personality assessments. В связке с Olivia: кандидат проходит скрининг + оценку прямо в чате, без перехода на сторонние платформы.

  • Что копировать: Inline assessments в чат-интерфейсе — квалификационные вопросы, ситуационные кейсы, проверка hard skills
  • Адаптация: Интеграция с российскими assessment-платформами (HRScanner, Talent Q, SHL Russia) или встроенные мини-тесты
  • AI-скоринг: Автоматическая оценка ответов с объяснением для рекрутера; человек всегда принимает финальное решение
  • Compliance: Уведомление кандидата об использовании AI (152-ФЗ согласие на обработку ПД + информирование об автоматизированных решениях)
Сложность: средняя Влияние: среднее Срок: 2-3 мес.

Модуль 3 — Skills Taxonomy & Internal Mobility (по модели Eightfold)

🎯

Talent Graph для российского рынка

Источник: Eightfold AI — 1.6B профилей, 1.6M навыков, deep learning

Eightfold построила крупнейший в мире talent graph на основе deep learning по карьерным траекториям. Для российского рынка аналога не существует — это возможность.

  • Что копировать: Skills-based подход к матчингу: не по должностям и резюме, а по навыкам и карьерным траекториям
  • Источники данных: hh.ru (миллионы резюме и вакансий), SuperJob, LinkedIn RU-сегмент, внутренние HR-данные клиентов
  • Skills ontology: Построить российскую таксономию навыков: IT-навыки (совместимы с глобальными) + специфичные RU-навыки (1С, российское право, гос. закупки)
  • Internal mobility: Рекомендации сотрудникам по карьерному развитию на основе их текущих навыков и целевых позиций внутри компании
  • Отличие от Eightfold: Не нужны 1.6B профилей — начать с корпоративных данных одного клиента (5K-50K сотрудников) + обогащение из открытых источников
Сложность: высокая Влияние: среднее (P2) Срок: 6-9 мес.
📊

AI Skills Assessment (по модели Workera)

Источник: Workera — adaptive testing, 10K+ skills, U.S. Air Force deployment

Workera использует computer-adaptive testing для точной оценки навыков. Каждый следующий вопрос зависит от ответа на предыдущий, обеспечивая точную калибровку за минимальное время.

  • Что копировать: Adaptive assessment engine — IRT-модель (Item Response Theory) для калибровки сложности вопросов
  • Применение: Входная оценка при найме, ежегодная аттестация, оценка после обучения, gap-анализ навыков команды
  • Контент: Банк вопросов для IT-навыков (Python, SQL, ML, DevOps) + soft skills + отраслевые компетенции
  • AI Mentor (как Sage у Workera): Персонализированные рекомендации по обучению на основе результатов оценки
Сложность: средняя Влияние: среднее Срок: 3-5 мес.

Модуль 4 — Employee-Facing Search + Action (по модели Moveworks)

🔍

Enterprise Search по HR-базе знаний

Источник: Moveworks Enterprise Search — 5.5M пользователей, 90% full deployment

Moveworks создал unified search interface, где сотрудник находит ответ на любой HR/IT вопрос через один поисковый запрос. Reasoning Engine понимает контекст и находит релевантную информацию из разных систем.

  • Что копировать: Единая точка входа «Спроси что угодно» — поиск по политикам, ЛНА, FAQ, прошлым тикетам, базе знаний
  • RAG Pipeline: Индексация корпоративных документов (Положения, Приказы, Инструкции, FAQ) + semantic search + LLM для генерации ответов с цитированием источника
  • Источники: 1С:Документооборот, Confluence/Wiki, SharePoint, корпоративный портал, регламенты
  • Контекст: Ответ учитывает роль сотрудника, подразделение, грейд, локацию — один вопрос «сколько у меня дней отпуска» даёт персонализированный ответ
Сложность: средняя Влияние: высокое Срок: 2-4 мес. MVP
⚡

Action Engine — выполнение действий через чат

Источник: Moveworks Agentic Automation + Leena AI Action Agents

Не только отвечать на вопросы, но и выполнять действия: подать заявление на отпуск, запросить справку, изменить данные. Moveworks достигает 50%+ ticket deflection именно потому, что «делает», а не только «отвечает».

  • Что копировать: Action framework — набор типовых действий, которые AI выполняет в backend-системах по запросу сотрудника
  • Действия для MVP: Подача заявления на отпуск (1С:ЗУП), запрос справки 2-НДФЛ, обновление персональных данных, заказ пропуска
  • Human-in-the-loop: Для критичных действий (увольнение, изменение оклада) — формирование черновика + маршрутизация на согласование HR
  • Безопасность: RBAC — сотрудник видит и может делать только то, что ему разрешено; audit trail каждого действия
Сложность: высокая Влияние: критичное Срок: 4-6 мес.

Критические GAP-ы — что никто не строит

On-Prem развёртывание

0 из 10 предлагают on-prem. Российский enterprise (гос., финансы, оборона) требует суверенности данных. Главный конкурентный моат.

Интеграция с 1С:ЗУП

Западные стартапы — Workday/SuccessFactors. Нужна нативная интеграция с 1С:ЗУП, БОСС-Кадровик, SAP HCM Russia.

Локальные LLM на своей инфраструктуре

Multi-LLM стек на on-prem: YandexGPT/GigaChat + Mistral/Llama. Без внешних API для персданных.

152-ФЗ + ФСТЭК

SOC 2 / GDPR не релевантны. Нужен 152-ФЗ compliance, ФСТЭК для гос. органов, локализация данных.

Русский HR NLP

Западные модели не понимают русскую HR-терминологию, ТК РФ, ЛНА. Fine-tune на трудовом кодексе и кадровых документах.

Skills taxonomy для RU рынка

Нет аналога talent graph Eightfold. Возможность построить на данных hh.ru, SuperJob, внутренних HR-систем.
